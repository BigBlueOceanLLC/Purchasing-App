üö® The Bug

Issue: When entering an estimated arrival date of August 14th in the shipment form, it appeared on the calendar as August 13th.

Root Cause:

The problem was in how HTML date input values were being parsed and converted to JavaScript Date objects.

‚ùå Input: "2024-08-14" ‚Üí new Date("2024-08-14") ‚Üí August 13th

The issue occurred because:

  • new Date("2024-08-14") creates a Date at midnight UTC
  • If you're in a timezone west of UTC (like PDT/PST), this becomes the previous day locally
  • The calendar then displayed the local date, which was one day earlier

‚úÖ The Fix

Solution: Added proper timezone-aware date parsing utilities that ensure dates stay in local timezone.

‚úÖ Input: "2024-08-14" ‚Üí parseDateInput("2024-08-14") ‚Üí August 14th

New Utility Functions Added:

// Safely parse HTML date input in local timezone export const parseDateInput = (dateString: string): Date => { const [year, month, day] = dateString.split('-').map(Number); return new Date(year, month - 1, day); // month is 0-indexed }; // Format Date for HTML date input export const formatDateForInput = (date: Date): string => { const year = date.getFullYear(); const month = String(date.getMonth() + 1).padStart(2, '0'); const day = String(date.getDate()).padStart(2, '0'); return `${year}-${month}-${day}`; };
